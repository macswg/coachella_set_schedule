# Phase 1 — Foundation & Schema

**Issue:** [#31](https://github.com/macswg/coachella_set_schedule/issues/31) — closed
**Epic:** [#30](https://github.com/macswg/coachella_set_schedule/issues/30)
**Status:** ✅ done in commit `f0cdf15`
**Depends on:** nothing
**Blocks:** Phase 2

## Context

Bootstrap the SQLite data layer. No behavioral change for users yet — this phase just lands the schema, migration tooling, and engine plumbing so Phase 2 can implement the store. The app must continue to run in `sheets` and `memory` modes unchanged.

## Files to create / modify

- `requirements.txt` — add `sqlalchemy>=2.0` and `alembic>=1.13`.
- `app/db/__init__.py` — empty package marker.
- `app/db/engine.py` — `get_engine()`, `get_session()` factory using `SQLITE_PATH` from config (default `./data/schedule.db`).
- `app/db/models.py` — SQLAlchemy declarative models:
  - `Show(id, name, position, is_current, is_previous, created_at)` — `name` unique; `is_current`/`is_previous` are booleans with DB-level check that at most one row has each.
  - `Act(id, show_id FK, row_index, act_name, scheduled_start, scheduled_end, actual_start, actual_end, screentime_total_seconds, screentime_session_start)` — times stored as `TIME` (naive, festival local).
  - `RecordingEvent(id, show_id FK, act_id FK nullable, event_type, fired_at)` — append-only log of trigger firings; useful for Phase 7 analytics.
- `alembic.ini` — generated by `alembic init`, edited to point `script_location = alembic` and read DB URL from `app/db/engine.py`.
- `alembic/` — generated scaffold; `alembic/env.py` imports `app/db/models.py` metadata.
- `alembic/versions/0001_initial.py` — creates all three tables.
- `main.py` — new startup hook runs `alembic upgrade head` (or `Base.metadata.create_all` guarded by env check) only when `DATA_BACKEND` is `sqlite`.
- `app/config.py` — add `SQLITE_PATH` setting.
- `docker-compose.yml`, `docker-compose.dev.yml`, `docker-compose.linux-dev.yml` — add a `./data:/app/data` volume mount so the `.db` file survives container rebuild.
- `.gitignore` — add `data/*.db` and `data/*.db-journal`.

## Key decisions

- **SQLAlchemy 2.x** with typed declarative models — modern, widely supported, and the only extra runtime dep.
- **Alembic** for migrations — real migrations matter once the schema ships and evolves across phases. On boot, run `alembic upgrade head` programmatically so there's no manual step.
- **Time storage** — keep naive `TIME` columns to match `app/models.py` semantics (midnight rollover is handled by the app, not the DB).
- **Screentime** — stored on `Act` to match the current Sheets layout (col H); running sessions use the same `screentime_session_start` pattern.

## Acceptance criteria

- `pip install -r requirements.txt` succeeds.
- `DATA_BACKEND=sqlite uvicorn main:app` starts cleanly on a fresh checkout, creates `./data/schedule.db`, and applies all migrations.
- `sqlite3 ./data/schedule.db ".schema"` shows `shows`, `acts`, `recording_events` tables.
- `DATA_BACKEND=sheets uvicorn main:app` and `DATA_BACKEND=memory uvicorn main:app` continue to work — no DB init runs.
- `docker compose up --build` mounts the `./data` volume and the DB file persists across `docker compose down && docker compose up`.
- `python -m pytest` passes (no new store tests yet; existing suite unaffected).

## Verification

```bash
# fresh install
pip install -r requirements.txt

# sqlite backend boots and creates schema
rm -f data/schedule.db
DATA_BACKEND=sqlite uvicorn main:app --port 8000 &
sleep 2
sqlite3 data/schedule.db ".schema"
# expect: shows, acts, recording_events tables

# other backends unaffected
DATA_BACKEND=sheets uvicorn main:app --port 8001 &   # boots without touching DB
DATA_BACKEND=memory uvicorn main:app --port 8002 &

# migration replay works
alembic downgrade base && alembic upgrade head
```
